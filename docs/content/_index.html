A software mental model journey.

{{% /blocks/cover %}} {{% blocks/lead color="primary" %}} A good architecture design is fundamental to build a stable, scalable and sustainable software system. Documenting this architecture such way that we provide required level of information to all stakeholders had always been a challenge. Thus, C4 model had been introduced as an abstraction-first approach for sketching architecture. In addition, to easily maintain and keep this documentation up to date, a second approach, named architecture as a code, enables to document the architecture design based on textual model. The combination of both brings a robust strategy for documenting and maintain any architecture design. {{% /blocks/lead %}} {{% blocks/section color="dark" type="row"%}} {{% blocks/feature icon="fa-layer-group" title="Ubiquitous language" %}} Design and read a system architecture documentation based upon C4 model. {{% /blocks/feature %}} {{% blocks/feature icon="fa-code" title="Code-first approach" %}} Use a textual Domain Specific Language leveraged by Structurizr to document an overall architecture as a code. {{% /blocks/feature %}} {{% blocks/feature icon="fa-screwdriver-wrench" title="Powerful toolchain" %}} Use a set of tools, namely Cornifer & Structurizr, to easily design an overall architecture. {{% /blocks/feature %}} {{% /blocks/section %}} {{% blocks/lead color="primary" %}} During our journey, we stressed all the benefits one could expect from moving towards architecture as code or code-first approach. However, most of the journey focused on what one could expect to gain from a single model defined as code, without dwelling on how authoring this model for too long. It is now time to address this point... {{% /blocks/lead %}}
